Posted on 2021-03-30

이진 탐색 트리 (Binary Search Tree)

이진 탐색 트리에는 값에 ‘크기’ 혹은 ‘크고 작음’ 이 있어서 각 노드를 기준으로 큰 값을 오른쪽 작은 값을 왼쪽으로 보내서 정렬하는 트리 구조입니다.

이진 탐색 트리는 밸런스가 잘 맞게 들어간 경우 자료의 탐색, 삽입, 그리고 삭제 모두 O(logN)의 시간 복잡도를 가지게되는 장점이 있지만 트리에 자료를 넣는 순서에 영향을 받고 자료의 삭제가 빈번해지다 보면 Scewed Tree가 될 가능성이 높아서 성능이 하락할수 있습니다.

이때 최악의 경우에는 자료의 탐색, 삽입, 삭제에 리스트 구조와 같은 O(N)의 시간 복잡도를 가지게 되어 성능이 많이 하락하게 됩니다.

정리하자면 이진 탐색 트리의 자료의 탐색, 삽입, 그리고 삭제의 시간복잡도는

**O(logN) <= 시간 복잡도 <= O(N)**가 됩니다.

자료 탐색

우선 루트 노드로 들어가 값을 확인하고 탐색하고자 하는 값과 비교하여 크다면 오른쪽 작다면 왼쪽으로 반복적으로 들어갑니다.

반복적으로 값을 비교하여 원하는 값이 나올때까지 쭉 들어갑니다.

모든 값을 비교할 필요 없이 크고 작은 값으로 비교하여 진행하기에 균형이 좋은 트리일수록 O(logN)에 가까운 시간 복잡도로 작동합니다.

자료의 삽입

탐색과 같은 과정을 진행하지만 끝까지 값을 비교하여 더 이상 비교할 값이 없어지게 되면 상위 노드와 값을 비교하여 크다면 오른쪽 작다면 왼쪽으로 값을 삽입합니다.

중요 - 이진 탐색 트리에서는 동일한 값이 존재한다면 삽입을 진행하지 않습니다.

자료의 삭제

자료의 삭제는 이진 탐색 트리에서 경우의 수가 매우 많아 복잡한 기능인데요!

우선 크게 세 가지의 상황으로 분류됩니다.

  1. 첫 번째로는, 자식 노드가 없는 경우
    • 이때는 그냥 상위 노드에서 해당 노드의 방향의 엣지를 끊어주면 됩니다.
  2. 두 번째로는, 자식이 하나 있는 경우
    • 이때는 자식 노드를 위로 올린다는 생각으로 상위 노드와 연결 시켜줍니다.
  3. 세 번째로는, 자식이 모두 있는 경우 (문제의 상황…..)자식이 양쪽 모두에 있는 노드를 삭제할때는 삭제할 노드를 대체할 자식 노드를 찾아줘야 하는데요
    • 이때 사용 할 수 있는 방법으로는 왼쪽 노드로 들어간 후 가장 우측에 있는 값 (즉, 왼쪽 노드에서 가장 큰 값)
      오른쪽 노드로 들어간 후 가장 좌측에 있는 값 (즉, 오른쪽 노드에서 가장 작은 값)으로 노드를 대체해 주는것이 있습니다.
      다만, 이때 그냥 그 값을 대체해 주는게 아니라 연결하는 방향 및 그 자식 노드들의 연결이 끊어지지 않게 구현하는게 중요해 다양한 예외 케이스 처리를 해줘야 하는게 중요합니다!
